1 Documentation

interface

data-provider<%> : interface?

The data-provider<%> interface provides access to a data layer.

method

(send a-data-provider stops) → (listof stop?)

Returns all stops. Since these are never changed, caching can happen.

method

(send a-data-provider stops-by-id) → (hash/c number? stop?)

Returns all stops in a hashset with stop-id as their key. Caching may happen.

method

(send a-data-provider routes) → (listof route?)

Returns all routes. Since routes may change, use add-callback to register for changes.

method

(send a-data-provider routes-for-stop stop) → (listof route?)

  stop : stop?
Returns all routes that use the given stop.

method

(send a-data-provider route-exists? route) → boolean?

  route : route?
Takes a route structure and checks whether it already exists in the data layer. route’s route-id is ignored.

method

(send a-data-provider insert-route stop    
  stop-ids) → void
  stop : stop?
  stop-ids : (listof number?)
Inserts a new route.

stop-ids is optional and defaults to null. If given, all stops referenced in stop-ids are linked with the new route.

method

(send a-data-provider add-callback callback) → void

  callback : (void . -> . void)
Adds a callback to the data layer, which is invoked if the routes have changed.
